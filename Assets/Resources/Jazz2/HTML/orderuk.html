Ordering Online

With a normal connection (http://), data transfered over the web can potentially be read by any computer it passes through. To combat this problem, certain Web sites offer secured connections (https://) by encrypting all data sent between the client and server. This system relys on the use of two encryption keys, one public and one private. The public key is used by the client browser to encrypt data sent to the server as well as decrypt any information that is returned. The private key works in the same manner and is secured for use by the server only. Only the public key can decrypt data sent by the private key and vice versa. Individually, the keys are useless. However, when employed properly, they ensure safe data transfers between the client and server. Because the private key is only available to the server, your data can not be decrypted or read by anyone else but the server.

When you have a secure connection to our server, your browser will let you know. Netscape users will see an icon of a key in the lower left-hand corner of the browser. Microsoft Internet Explorer will display an icon of a pad lock in the lower right-hand corner. You will also notice that the URL says https instead of http. Because these order forms are stored on your local hard disk, you will not see these icons straight away. Although you will have a secure connection, these icons will appear near the end of your transaction.
